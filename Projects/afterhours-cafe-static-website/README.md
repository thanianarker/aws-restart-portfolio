# After Hours Café Static Website (Amazon S3)

## 📌 Project Overview

This project demonstrates how a small café can improve its online presence by hosting a **static website using Amazon S3**.

The website provides customers with essential information about the café, including the menu, business description, and contact details. By using AWS, the café can host its website in a scalable, reliable, and cost-effective way without managing traditional web servers.

---

## 🏪 Business Scenario

After Hours Café is a local business that currently relies mainly on **on-premises systems** for daily operations and has limited digital presence.

Challenges include:

- Limited online visibility  
- Dependence on local systems  
- Risk of downtime or data loss  
- Difficulty scaling digital services as the business grows  

To address these issues, the café plans to begin adopting **cloud solutions using AWS**.

---

## ☁️ Proposed Cloud Solution

The café website is hosted using **Amazon S3 Static Website Hosting**.

This solution allows the business to:

- Host a website without managing servers  
- Deliver reliable access to customers online  
- Store website files securely  
- Scale automatically as website traffic grows  

---

## 🧰 AWS Service Used

### Amazon S3 (Simple Storage Service)

Amazon S3 is used to host the static website.

Key benefits include:

- **Highly Scalable** – Can handle growing website traffic automatically  
- **Highly Durable** – 99.999999999% durability for stored data  
- **Cost-Effective** – Pay only for storage and usage  
- **Static Website Hosting** – Supports direct hosting of HTML, CSS, JavaScript, and images  

---

## 🌐 Website Features

The website includes:

- Café name: After Hours Café  
- Description of the business  
- Menu with at least 5 items  
- Contact information for customers  

---

## ⚙️ Implementation Steps

1. Created an **Amazon S3 bucket**  
2. Uploaded website files (HTML, CSS, images)  
3. Enabled **Static Website Hosting**  
4. Configured bucket permissions for public access  
5. Accessed the website through the **S3 website endpoint**
